# 📚 누구나 쉽게 이해하는 디지털 교과서 플랫폼 코드 설명서

이 문서는 **"고등학생도 이해할 수 있는"** 수준으로 작성된 **디지털 교과서 플랫폼(Digital Textbook Platform)** 의 코드 설명서입니다. 
이 프로젝트가 어떻게 구성되어 있고, 각 코드가 어떤 역할을 하는지 아주 쉽게 풀어서 설명해 드릴게요!

---

## 🏗️ 1. 전체적인 구조: 집 짓기에 비유하기

웹사이트를 만드는 것은 **집을 짓는 것**과 아주 비슷합니다. 이 프로젝트는 크게 세 가지 핵심 요소로 이루어져 있습니다.

1.  **HTML (`index.html`) - 집의 뼈대**: 방은 어디에 있고, 문은 어디에 달지 구조를 잡는 역할입니다.
2.  **CSS (`styles/main.css`) - 인테리어**: 벽지는 무슨 색으로 할지, 가구는 어떻게 배치할지 예쁘게 꾸미는 역할입니다.
3.  **JavaScript (`scripts/main.js`) - 전기와 가전제품**: 스위치를 누르면 불이 켜지고, 리모컨으로 TV를 켜듯이 사용자와 상호작용하여 실제로 무언가 작동하게 하는 역할입니다.

그리고 **백엔드(API)** 라는 친구가 하나 더 있는데, 이건 **외부와 소통하는 전화기**라고 생각하면 됩니다. (AI에게 질문을 보내고 답을 받아오죠!)

---

## 📂 2. 파일들이 하는 일 (파일 구조)

이 폴더 안에 있는 파일들이 각각 무슨 일을 하는지 살펴볼까요?

*   **`index.html`**: 이 프로그램의 **대문**입니다. 인터넷 브라우저가 가장 먼저 읽는 파일로, 전체적인 화면 구성을 담당합니다.
*   **`styles/main.css`**: 화면을 **예쁘게** 만들어주는 스타일 파일입니다. 글자 크기, 색상, 배치 등을 담당합니다.
*   **`scripts/main.js`**: 화면의 **버튼을 눌렀을 때 무슨 일이 일어날지** 정해주는 행동 대장입니다. (탭 전환, 채팅 보내기 등)
*   **`api/coze-chat.js`**: **AI 튜터와 대화**하기 위해 질문을 배달하고 답장을 받아오는 "우체부" 역할을 합니다.
*   **`chapter_01_part2.pdf`**: 실제 교과서 내용이 담긴 PDF 파일입니다.
*   **`easy_explanation.html` / `practice_quiz.html`**: '쉬운 해설'과 '퀴즈' 탭을 눌렀을 때 보여줄 별도의 페이지들입니다.

## 🧐 3. 상세 코드 설명과 동작 원리

이제 각 파일의 코드가 어떻게 작동하는지 자세히, 그리고 쉽게 알아봅시다.

### (1) `index.html` - 화면 구성하기

이 파일은 크게 **머리(Head)** 와 **몸통(Body)** 으로 나뉩니다.

*   **`<head>` (머리)**: 화면에는 안 보이지만 중요한 설정을 합니다.
    *   글씨체(폰트)를 가져오고, `styles/main.css`를 불러와서 "이 스타일대로 꾸며줘!"라고 명령합니다.
*   **`<body>` (몸통)**: 실제 눈에 보이는 부분입니다. 크게 3등분 되어 있습니다.
    1.  **왼쪽 (`sidebar-left`)**: **목차**가 있습니다. 1장, 2장... 클릭해서 이동할 수 있죠.
    2.  **가운데 (`main-viewer`)**: **진짜 공부하는 공간**입니다. 
        *   `<iframe>` 태그를 사용하는데, 이건 **"웹페이지 안의 또 다른 작은 창"** 입니다. 이 창을 통해 PDF도 보여주고, 퀴즈도 보여줍니다.
    3.  **오른쪽 (`sidebar-right`)**: **도우미 공간**입니다. 메모장, AI 튜터 채팅창, 용어 사전이 있습니다.

### (2) `scripts/main.js` - 움직임 만들기

이 파일이 있어야 버튼이 동작합니다. 중요 기능들을 하나씩 볼까요?

#### 🌟 탭 전환 기능 (`initMainTabs`, `initPanelTabs`)
*   **원리**: 사용자가 '쉬운 해설' 버튼을 클릭하면?
    1.  컴퓨터는 모든 화면(`article`)을 일단 **숨깁니다** (`hidden` 클래스 추가).
    2.  그리고 사용자가 클릭한 그 화면만 **보여줍니다** (`active` 클래스 추가).
    3.  마치 여러 장의 카드를 겹쳐 놓고, 원하는 카드만 맨 위로 올리는 것과 같습니다.

#### 🌟 목차 이동 기능 (`initTOC`)
*   **원리**: 왼쪽 목차에서 '1장'을 클릭하면?
    1.  `switchChapter(1)`이라는 명령어가 실행됩니다.
    2.  화면의 진행 바(Progress Bar)를 25%로 채웁니다.
    3.  AI 튜터가 "1장에 대해 궁금한 게 있나요?"라고 인사말을 바꿉니다.

#### 🌟 AI 채팅 기능 (`initChat`, `sendMessage`)
*   **원리**: 채팅창에 질문을 쓰고 전송 버튼을 누르면 일어나는 일입니다.
    1.  여러분이 쓴 글자를 가져와서 채팅창에 **말풍선**으로 붙입니다.
    2.  잠시 "생각 중..." 이라는 점 3개 애니메이션을 보여줍니다.
    3.  **`fetch`** 라는 명령어를 사용해, 질문을 `api/coze-chat`이라는 **우체국**으로 보냅니다.
    4.  우체국에서 답장이 오면, 점 3개를 지우고 **AI의 답변 말풍선**을 붙여줍니다.

### (3) `api/coze-chat.js` - AI와 소통하기 (백엔드)

이 파일은 사용자의 브라우저(크롬 등)가 아니라, **서버(서비스를 제공하는 컴퓨터)** 에서 실행되는 코드입니다.

*   **역할**: 여러분의 질문을 받아서 **진짜 똑똑한 AI(Coze API)** 에게 전달하고, AI가 해준 답변을 다시 여러분에게 돌려줍니다.
*   **보안관**: AI를 쓰려면 `Token`(비밀 열쇠)이 필요한데, 이 열쇠는 소중하니까 사용자에게 안 보여주고 이 파일 안에서만 몰래 사용합니다.
*   **동작 과정**:
    1.  질문 도착! ("ESG가 뭐야?")
    2.  미리 설정된 **시스템 규칙(System Prompt)** 을 추가합니다. ("너는 친절한 선생님이야. 학생 수준에 맞춰 설명해줘.")
    3.  진짜 AI에게 전송! (질문 + 규칙 + 비밀 열쇠)
    4.  AI의 답변 도착! ("ESG는 환경, 사회, 지배구조를 뜻해요.")
    5.  다시 사용자에게 답변 전달!

## 🚀 4. 실행 방법 (내 컴퓨터에서 켜보기)

이 프로젝트를 여러분의 컴퓨터에서 직접 실행해보고 싶다면 이렇게 하세요!

1.  **폴더 열기**: 탐색기에서 이 프로젝트 폴더를 엽니다.
2.  **서버 켜기**:
    *   그냥 `index.html`을 더블 클릭해도 열리지만, 채팅 기능 등이 제대로 안 될 수 있어요.
    *   **VS Code**를 쓴다면 `Live Server` 확장 기능을 켜는 것이 가장 좋습니다.
    *   또는 파이썬이 있다면 터미널에 `python -m http.server 8000`을 입력해보세요.
3.  **접속하기**: 인터넷 주소창에 `http://localhost:8000`을 입력하면 멋진 교과서 플랫폼이 나타납니다!

## 📝 5. 용어 정리

*   **프론트엔드 (Frontend)**: `HTML`, `CSS`, `JS`처럼 사용자가 눈으로 보고 클릭하는 화면을 만드는 기술입니다.
*   **백엔드 (Backend)**: `api/` 폴더처럼 눈에 안 보이지만 뒤에서 데이터를 처리하고 서버와 통신하는 기술입니다.
*   **API**: 프로그램끼리 대화하는 약속입니다. "A라고 물어보면 B라고 대답해줄게"라는 규칙이죠.
*   **DOM (문서 객체 모델)**: 자바스크립트가 HTML을 조작하기 위해 사용하는 도구입니다. "HTML 태그를 잡아서 내용을 바꿔라!" 같은 명령을 내릴 때 씁니다.

이 설명서가 있다면 여러분도 이 코드를 이해하고 수정할 수 있을 거예요! 화이팅! 🌱
