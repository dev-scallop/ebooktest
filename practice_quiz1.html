문제 1. 파이썬의 주석과 코드 작성법에 대한 설명으로 옳지 않은 것은?

본문에서 설명한 파이썬 프로그램의 기본 작성법과 주석 처리에 대한 내용 중 틀린 것을 고르시오.
  1. 한 줄 주석을 달 때는 코드 앞에 # 기호를 사용한다.
  2. 여러 줄에 걸쳐 주석을 달고 싶다면 작은따옴표 3개(''')를 앞뒤로 감싸서 사용한다.
  3. 코드가 너무 길어져서 줄을 바꾸어 쓰고 싶을 때는 문장의 맨 뒤에 /(슬래시)를 붙인다.
  4. 프로그램의 가독성을 위해 함수 선언, 변수 선언, 메인 코드 부분으로 나누어 작성하는 것이 권장된다.
정답 및 해설 확인하기

정답: 3번

해설: 파이썬에서 코드가 길어져 줄을 바꿀 때, 컴퓨터가 한 줄로 인식하게 하려면 문장 맨 뒤에 역슬래시(\)를 붙여야 합니다. 슬래시(/)는 나눗셈 연산자입니다.

[객관식] 난이도: 중

문제 2. 파이썬과 컴파일러 언어(C/C++, 자바)의 변수 선언 차이점으로 올바른 것은?

본문의 '여기서 잠깐 - 변수의 선언' 내용을 바탕으로 올바른 설명을 고르시오.
  1. 파이썬은 변수를 사용하기 전에 반드시 자료형(int, float 등)을 명시하여 선언해야 한다.
  2. C언어는 변수를 선언하지 않고 바로 값을 대입하여 사용할 수 있다.
  3. 파이썬은 a = 100과 같이 값을 대입하는 순간 변수가 자동으로 선언 및 생성된다.
  4. 파이썬에서는 변수 선언 부분에 초깃값을 미리 대입해 두는 것이 문법적으로 금지되어 있다.
정답 및 해설 확인하기

정답: 3번

해설: 파이썬과 같은 인터프리터 언어는 변수를 별도로 선언하지 않고, 값을 대입하는 시점에 변수가 자동 생성되며 자료형이 결정됩니다. 반면 C/C++, 자바 등은 사용 전 반드시 변수 선언이 필요합니다.

[객관식] 난이도: 중

문제 3. 터틀 그래픽 함수에 대한 설명으로 적절하지 않은 것은?

다음 중 본문에서 사용된 터틀 그래픽 관련 함수들의 기능 설명으로 틀린 것은 무엇인가?
  1. turtle.penup(): 펜을 들어 올려 이동 시 선이 그려지지 않게 한다.
  2. turtle.goto(x, y): 거북이를 좌표 (x, y) 위치로 이동시킨다.
  3. turtle.onscreenclick(함수명, 1): 마우스 오른쪽 버튼을 클릭했을 때 지정된 함수를 실행한다.
  4. turtle.shapesize(크기): 거북이의 크기를 설정된 값으로 변경한다.
정답 및 해설 확인하기

정답: 3번

해설: turtle.onscreenclick(함수명, 번호)에서 번호가 1이면 마우스 '왼쪽' 버튼, 2는 '가운데' 버튼, 3은 '오른쪽' 버튼을 의미합니다. 따라서 3번 설명은 틀렸습니다.

[단답형] 난이도: 하
